# LaiprangBrain & GeminiSearch — Freestyle Zone

สร้างและควบคุมโดย: **แชมป์ (James Suersuwan)** ร่วมกับ **ClaudeMD**

เว็บแอปศูนย์รวมโซเชียลมีเดีย (TikTok / IG / FB / X) สำหรับวัยรุ่น
กฎเหล็กข้อเดียว: **ห้ามทำให้ใครเดือดร้อนเป็นอันขาด**

## วิธีรัน

1. ติดตั้ง Node.js (v18+) ก่อน
2. ก็อปไฟล์ตั้งค่า:
   ```
   cp .env.example .env
   ```
3. เปิด `.env` แล้วใส่ค่าจริง:
   - `SESSION_SECRET` — สุ่มสตริงยาวๆ
   - API key ของแต่ละแพลตฟอร์ม (TikTok/IG/FB/X) — ถ้ายังไม่มี ปล่อยว่างไว้ก่อนได้ ระบบจะขึ้นว่า "ยังไม่ได้ตั้งค่า"
   - `GEMINI_API_KEY` — ใช้ให้ AI พี่น้อง (BigBrother) วิเคราะห์เชิงลึก ถ้าไม่ใส่ ระบบจะใช้แค่ LittleBrother (heuristic scan) แทน
4. ติดตั้ง dependency:
   ```
   npm install
   ```
5. รันเซิร์ฟเวอร์:
   ```
   npm start
   ```
6. เปิดเบราว์เซอร์ที่ `http://localhost:3000`

## โครงสร้างไฟล์

- `server.js` — Express server, Helmet security headers, rate limiting, ทุก API route + WebSocket แชทไลฟ์
- `membership.js` — ระบบสมาชิก 3 ระดับ (Standard/Plus/Premium โปรโมทตัวเองได้ทุกระดับ ต่างกันที่โควต้า) + VIP Ultimate (โควต้าไม่จำกัด)
- `aiGuardian.js` — AI พี่น้อง (LittleBrother = heuristic, BigBrother = heuristic + Gemini) พร้อมเก็บสถิติ/ประวัติสำหรับ dashboard รายงานผล
- `liveShop.js` — จัดการห้องไลฟ์ขายของ (สินค้าปักหมุด + เก็บ log แชทสั้นๆ)
- `public/` — หน้าเว็บ (index.html, style.css, app.js) มี 5 แท็บ: ศูนย์รวม / โพสต์ใหม่ / ไลฟ์ขายของ / AI พี่น้อง (รายงานผล) / สมาชิก
- `.env.example` — เทมเพลตค่าตั้งค่า (คัดลอกเป็น `.env` แล้วใส่ค่าจริง ห้าม commit ขึ้น git)

## ฟีเจอร์ไลฟ์ขายของ

- กด "เริ่มไลฟ์" → เปิดห้อง → แชทสด (WebSocket) ทุกข้อความผ่าน AI พี่น้องสแกนก่อนกระจายเสมอ
- เจ้าของไลฟ์ปักหมุดสินค้า (ชื่อ/ราคา/ลิงก์) ให้คนดูเห็นระหว่างไลฟ์
- เนื้อหาการซื้อขาย/การตกลงกันเองในแชท เราไม่เข้าไปยุ่ง สแกนเฉพาะภัยคุกคาม (ลิงก์ฟิชชิ่ง/ข้อความหลอกลวง) เท่านั้น
- **วิดีโอสดจริง (ภาพ/เสียง) ยังไม่ได้ต่อ** — ช่อง `<video>` ในหน้าเป็นแค่ placeholder ต้องเสียบ SDK ของผู้ให้บริการสตรีมมิ่งจริง เช่น Agora, LiveKit, หรือ Mux เข้ามาเอง (การเขียน WebRTC/RTMP encoder เองทั้งระบบไม่ใช่อะไรที่ทำในไฟล์เดียวได้ ต้องใช้บริการเหล่านี้ช่วย)

## AI พี่น้อง — หน้ารายงานผล

แท็บ "AI พี่น้อง" ตอนนี้มี:
- สถิติสแกนวันนี้ (สแกนทั้งหมด / บล็อกความเสี่ยงสูง / น่าสงสัย)
- กราฟแท่งแยกตามประเภทความเสี่ยงที่เจอ (คำต้องสงสัย / ลิงก์เสี่ยง / ลิงก์เยอะผิดปกติ / รูปแบบพฤติกรรมล่อลวง)
- ตารางประวัติการสแกนล่าสุด 20 รายการ
- ตารางรายชื่อผู้ใช้ที่ถูกส่งให้แอดมินตรวจสอบ (ทำผิดซ้ำหรือเสี่ยงร้ายแรง)

## ระบบตรวจจับพฤติกรรมเสี่ยงล่อลวงเด็ก + การยกระดับอัตโนมัติ (ใหม่)

- `aiGuardian.js` เพิ่มการตรวจจับ **รูปแบบพฤติกรรม** (ไม่ใช่คลังคำพูดสำเร็จรูป) ที่เข้าข่ายล่อลวง เช่น ชวนย้ายไปคุยแอปอื่นแบบลับๆ, ขอให้ปิดเป็นความลับจากพ่อแม่, ขอรูป/ข้อมูลส่วนตัวไม่เหมาะสม — ให้น้ำหนักคะแนนสูงสุดเพราะผลกระทบร้ายแรงกว่าฟิชชิ่งทั่วไป
- `safetyTracker.js` (ใหม่) สะสมประวัติความเสี่ยงรายคน แล้วยกระดับอัตโนมัติ:
  - ครั้งที่ 1 → แจ้งเตือน (warn)
  - ครั้งที่ 3 → มิวท์ชั่วคราว 15 นาที (แชท/โพสต์ไม่ได้ชั่วคราว)
  - ครั้งที่ 5 ขึ้นไป หรือเจอรูปแบบล่อลวงแม้แค่ครั้งเดียว → ส่งให้แอดมิน/มนุษย์ตรวจสอบทันที (`flagged_for_review`) — **ระบบจะไม่ตัดสิน/แบนใครแบบถาวรเองอัตโนมัติ** เพราะเรื่องระดับนี้ควรให้คนเป็นคนตัดสินใจสุดท้ายเสมอ
- ดูรายชื่อที่ถูก flag ได้ที่แท็บ "AI พี่น้อง" หรือยิง `GET /api/guardian/flagged` ตรง (ควรจำกัดสิทธิ์เฉพาะแอดมินก่อนขึ้นจริง — ตอนนี้ยังไม่มีระบบสิทธิ์แอดมิน)
- **ข้อจำกัดที่ต้องรู้**: นี่คือ heuristic เริ่มต้นเท่านั้น ตรวจจับได้เฉพาะรูปแบบที่ฝังไว้ล่วงหน้า คนที่ตั้งใจเลี่ยงสามารถหลบได้ ไม่ควรใช้แทนการมีทีมดูแลความปลอดภัยที่เป็นมนุษย์จริง โดยเฉพาะแพลตฟอร์มที่มีเด็กใช้งาน

## ติดตามพฤติกรรม (Behavior Baseline) — ชั้นที่ 3 ของ AI พี่น้อง

`behaviorBaseline.js` (ใหม่) เพิ่มการตรวจจับแบบ **behavior-based** ต่อยอดจาก signature/pattern-based เดิม ตามแนวคิดที่ว่าไวรัส/มัลแวร์สมัยใหม่ (เช่น dropper apps, dynamic code loading) มักหลบการสแกน signature ได้ง่าย แต่หลบพฤติกรรมผิดปกติได้ยากกว่า:

- **burst_activity** — โพสต์/แชทถี่ผิดปกติ (≥8 ครั้งใน 5 นาที)
- **rate_spike** — อัตราการโพสต์พุ่งขึ้นกว่า baseline ปกติของคนนั้นเอง (เทียบกับค่าเฉลี่ยย้อนหลัง 2 ชั่วโมง)
- **new_domain_pattern** — จู่ๆ เริ่มแชร์ลิงก์จากโดเมนใหม่ที่ไม่เคยใช้มาก่อน หลังจากมี pattern การใช้โดเมนเดิมมาสักระยะ

**สำคัญมาก**: โมดูลนี้เป็น **advisory เท่านั้น** — แค่ขึ้นสัญญาณเตือนในหน้ารายงานผล (`GET /api/guardian/stats` → `behaviorAnomalies`) **ไม่มีสิทธิ์มิวท์/บล็อก/ลงโทษใครโดยอัตโนมัติเด็ดขาด** เพราะระบบความปลอดภัยอาศัยระบบอัตโนมัติล้วนๆ ไม่ได้ ต้องมีคนเป็นคนตัดสินใจขั้นสุดท้ายเสมอ

## จุดที่ต้องทำต่อก่อนขึ้นจริง (production)

ส่วนนี้คือของจริงที่ยังไม่มีใครใส่ให้ได้แทน เพราะต้องใช้บัญชี/สิทธิ์ของมึงเอง:

1. **OAuth ของแต่ละแพลตฟอร์ม** — proxy endpoint (`/api/proxy/...`) มีตัวอย่างของ X, YouTube, LINE ให้ดูเป็นแพทเทิร์นไว้แล้ว ต้องไปสมัคร Developer account ของ TikTok/IG/FB เพิ่มเพื่อได้ client key มาใส่ใน `.env` แล้วเพิ่ม route แบบเดียวกัน
2. **วิดีโอไลฟ์จริง** — ต้องสมัครบริการสตรีมมิ่ง (Agora/LiveKit/Mux) แล้วเสียบ SDK ฝั่ง client เข้าที่ `.live-video-placeholder` ใน `public/index.html`
3. **ฐานข้อมูลจริง** — ตอนนี้ `membership.js`, `liveShop.js`, และสถิติ Guardian เก็บใน memory (ข้อมูลหายเมื่อ restart เซิร์ฟเวอร์) โปรดักชันจริงต้องย้ายไป Postgres/MongoDB
4. **ระบบล็อกอินจริง** — ตอนนี้ auth เป็น mock (สุ่ม guest ID ผ่าน cookie) ต้องต่อ OAuth login จริงของแต่ละแพลตฟอร์ม หรือระบบสมัครสมาชิกของตัวเอง
5. **Google Safe Browsing / VirusTotal** — `aiGuardian.js` เว้น hook `checkAgainstSafeBrowsing()` ไว้ให้ต่อ threat-intel API จริงเพิ่มความแม่นยำ
6. **Payment gateway** — endpoint `/api/membership/upgrade` ตอนนี้แค่เปลี่ยน tier ทันทีไม่มีการเช็คจ่ายเงินจริง ต้องต่อ Omise/Stripe/2C2P ก่อนขึ้นจริง
